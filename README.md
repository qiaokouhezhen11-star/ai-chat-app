# AIチャットアプリ

ChatGPTライクなUIで、AIとの会話、ストリーミング応答、ファイル参照ができるチャットアプリです。  
`.txt` / `.md` ファイルをアップロードし、その内容をもとにAIへ質問できる機能を実装しています。

転職用ポートフォリオとして、**生成AIアプリの基本機能** に加えて、**実務を意識したファイル連携機能** まで実装しました。  
UIだけでなく、**ストリーミング応答・ファイルバリデーション・履歴管理** まで含めて設計、実装しています。

## 工夫したポイント

### 1. ファイル連携を自然にチャット体験へ組み込んだ
`.txt` / `.md` をアップロードし、その内容をAI回答に反映できるようにしました。  
また、**どのファイルを使って送信したかが履歴に残る設計** にしているため、会話をあとから見返したときにも分かりやすくしています。

### 2. 安全性を高めるためにフロントとサーバーの両方でファイルバリデーションを実施
未対応形式やPDFについて、**フロント側だけでなくサーバー側でも制御** しています。  
これにより、UI上の誤操作だけでなく、不正な送信に対しても安全な構成にしています。

### 3. UX向上のためにエラー表示を画面内UIに統一
初期段階では `alert` を使っていましたが、最終的には **画面内にエラーメッセージを表示する方式** に変更しました。  
これにより、ユーザー体験とアプリの完成度を高めています。

### 4. 状態管理のずれを避けるために再送処理を安定化
再送時に `setTimeout` に依存しないように整理し、**送る本文を直接渡せる構成** に変更しました。  
Reactのstate反映タイミングに依存しにくい、より安定した実装を意識しています。

### 5. 保守性を高めるために共通定数・表示ロジック・ファイル入力処理を分離
ファイル関連定数や表示用ヘルパー、ファイル入力処理を別ファイルへ切り出し、変更や拡張に強い構成に整理しました。

### 6. 停止後にワンクリックで再送できるよう改善
回答生成を途中で停止したあと、同じ質問を入力し直さなくても**ワンクリックで再送できるUI**を追加しました。  
また、停止状態が一目で分かるように表示も調整し、**状態表示と再操作が分かりやすいチャットUI**を意識して実装しています。

## セットアップ方法

依存関係をインストールします。

```bash
npm install
```

プロジェクト直下の `.env.example` を参考に、`.env.local` を作成してAPIキーを設定します。

```env
OPENAI_API_KEY=your_api_key_here
```

開発サーバーを起動します。

```bash
npm run dev
```

## 使い方

1. チャット画面でメッセージを入力します。  
2. 必要に応じて `.txt` / `.md` ファイルを添付します。  
3. 送信すると、AIがストリーミング形式で回答します。  
4. 途中で停止や再送もできます。  
5. 会話履歴はローカルに保存されます。
