# AIチャットアプリ

ChatGPTライクなUIで、AIとの会話、ストリーミング応答、ファイル参照ができるチャットアプリです。  
`.txt` / `.md` ファイルをアップロードし、その内容をもとにAIへ質問できる機能を実装しています。

転職用ポートフォリオとして、**生成AIアプリの基本機能** に加えて、**実務を意識したファイル連携機能** まで実装しました。  
UIだけでなく、**ストリーミング応答・ファイルバリデーション・履歴管理** まで含めて設計、実装しています。

## 工夫したポイント

### 今回の改善ポイント
- UIの見やすさを上げるため、空画面の案内カード、入力欄の操作ヒント、接続状態の表示を追加
- 会話管理をしやすくするため、会話一覧・検索・自動タイトル生成・切り替え・削除を実装
- メッセージ送信の安定性を上げるため、二重送信防止、停止後の再送、ファイル付き再送を実装
- APIの安全性を上げるため、受信メッセージの整形、履歴件数の制限、ファイルサイズの再検証を追加
- READMEで説明しやすいように、「UI」「安定性」「保守性」の3軸で改善内容を整理

### 1. ファイル連携を自然にチャット体験へ組み込んだ
`.txt` / `.md` をアップロードし、その内容をAI回答に反映できるようにしました。  
また、**どのファイルを使って送信したかが履歴に残る設計** にしているため、会話をあとから見返したときにも分かりやすくしています。

### 2. 安全性を高めるためにフロントとサーバーの両方でファイルバリデーションを実施
未対応形式やPDFについて、**フロント側だけでなくサーバー側でも制御** しています。  
これにより、UI上の誤操作だけでなく、不正な送信に対しても安全な構成にしています。

### 3. UX向上のためにエラー表示を画面内UIに統一
初期段階では `alert` を使っていましたが、最終的には **画面内にエラーメッセージを表示する方式** に変更しました。  
これにより、ユーザー体験とアプリの完成度を高めています。

### 4. 状態管理のずれを避けるために再送処理を安定化
再送時に `setTimeout` に依存しないように整理し、**送る本文を直接渡せる構成** に変更しました。  
Reactのstate反映タイミングに依存しにくい、より安定した実装を意識しています。

### 5. 保守性を高めるために共通定数・表示ロジック・ファイル入力処理を分離
ファイル関連定数や表示用ヘルパー、ファイル入力処理を別ファイルへ切り出し、変更や拡張に強い構成に整理しました。

### 6. 停止後の再操作をしやすいUIに改善
回答生成を途中で停止したあと、同じ質問を入力し直さなくても**ワンクリックで再送**できるUIを追加しました。  
また、停止した回答の文脈を引き継いで**続きを生成**できるようにし、チャットアプリとして自然な操作感を意識して実装しています。

さらに、同じページ内では**ファイル付き送信も再送できるように状態を保持**し、本文だけでなく添付ファイルも含めて再実行できるよう改善しました。

### 7. 会話履歴を一覧・検索・切り替えできるように改善
単一のチャット画面ではなく、**複数会話を保持して一覧表示できる構成**に拡張しました。  
また、会話タイトルの自動生成・検索・切り替え・削除まで対応し、実際のチャットプロダクトに近い操作感を意識して実装しています。

### 8. UIの使い方を画面内で理解しやすくした
初回表示時に、**このアプリで何ができるかを3つのカードで説明** するようにしました。  
さらに入力欄の上に、**選択中モデル・送信状態・対応ファイル形式** を表示し、READMEを読まなくても操作方法が分かる設計にしています。

### 9. 二重送信を防いで、送信まわりを安定化した
送信ボタン連打や Enter の連続入力で、**同じメッセージが重複送信されにくいロック処理** を追加しました。  
これにより、ストリーミング開始前の短いタイミングでも、送信状態がぶれにくい構成にしています。

### 10. API側でも入力を整えて壊れにくくした
フロントから渡されたメッセージをそのまま信用せず、**role / content を整形してから OpenAI API に渡す** ようにしました。  
また、履歴を直近20件に絞ることで、不要に大きなリクエストを避け、応答失敗のリスクを下げています。

## セットアップ方法

依存関係をインストールします。

```bash
npm install
```

プロジェクト直下の `.env.example` を参考に、`.env.local` を作成してAPIキーを設定します。

```env
OPENAI_API_KEY=your_api_key_here
```

開発サーバーを起動します。

```bash
npm run dev
```

## 使い方

1. チャット画面でメッセージを入力します。  
2. 必要に応じて `.txt` / `.md` ファイルを添付します。  
3. 送信すると、AIがストリーミング形式で回答します。  
4. 途中で停止や再送もできます。  
5. 会話履歴はローカルに保存されます。

## READMEに書きやすいアピール観点

### UI / UX
- ChatGPT風の2カラムUIをベースに、会話一覧・検索・自動タイトル生成を実装
- 空画面でも迷わないように、クイックアクションと操作ガイドを画面内に配置
- モデル選択、接続状態、対応ファイル形式をひと目で分かる形で表示

### 安定性
- ストリーミング中の停止、再送、続きを生成する操作を同じ画面で完結
- ファイル付き送信も同一画面内で再送できるよう状態を保持
- 二重送信防止ロックにより、連打や Enter 連続入力でも送信が重複しにくい
- フロントとサーバーの両方でファイル形式とサイズを検証

### 保守性
- 会話保存、タイトル生成、ファイル入力処理を役割ごとに分離
- UIロジックだけでなく、API側でも入力メッセージを整形して安全に処理
- READMEで機能紹介だけでなく、設計意図まで説明できる構成
